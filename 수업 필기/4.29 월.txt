05_평가지표

분류는 우리가 정답을 맞출 수 있다는 가정하에 시작함
회귀는 아님. 연속형과 수치형을 다루는데 이들은 값이 무한대가 될 수 있기에 정답이 정해져 있지 않다.
=> 그러므로 둘의 평가방법은 달라져야한다.

정밀도, ROC, AUC score는 이진분류에 특화되어있다.


Scikit-Learn은 분류, 회귀, 군집화 및 차원 축소 알고리즘과 같은 머신러닝에 필수적인 다양한 도구를 제공한다. 또한 데이터 정규화, 스케일링 및 인코딩을 위한 다양한 전처리 도구를 포함하고 있습니다.
데이터 분석 및 시각화에 필수적인 다른 널리 사용되는 파이썬 라이브러리인 NumPy, SciPy 및 Matplotlib 위에 구축되었습니다

정확도 평가지표의 문제 => 0과 1일떄 0일떄는 얼마나 맞았고 1일때는 얼마나 맞았는지가 궁금한데 확인할 수 없다.
근데 각각의 정확도가 중요한 경우가 많은데, 하나를 아예 못맞춰도 90프로의 정확도를 가질수있기때문에

X.shape  => (1797, 64)이고
X[0].shape => (64,) 인데 0번인덱스는 이미지 하나를 의미한다. 하지만 1차원인데 이는 8x8 이미지를 flatten한 것이다. 그래서 다시 reshape를 해줘서 이미지를 출력한다

TN : negative라 예측했는데 맞음
FN : negative라 예측했는데 틀림
FP : positive라 예측했는데 틀림
TP : positive라 예측했는데 맞음

recall/sensitivity => 실제 양성중에서 양성으로 예측한 비율
precision => 양성이라 예측한것중에서 실제 양성 비율
F1 => 위에 둘을 종합. 둘의 값이 비슷하면서 점수가 높은경우 더 높은 값을 가진다
> 정밀도와 재현율의 조화평균 점수


데이터 전처리(Preprocessing) 전: 모델링전 => 학습데이터를 모델이 학습하기 좋게 변환하는 작업

모델링 - 모델 수정 + 모델학습 + 검증

결과 후처리 (Postprocessing) 후: 모델추정한 후  -> 모델이 예측한 결과를 바꿔 정밀도를 올리거나 재현율을 올릴 수 있다.
하지만 하나를 올리면 하나는 떨어짐. 상충관계에 있다
