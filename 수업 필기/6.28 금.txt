view는 비즈니스 로직을 만든다.
만약에 로그인을 한다면 로그인이 비즈니스가 되는거고 로그인을 하는 과정을 비즈니스 로직이라고 한다

상속을 받았더라도 뭐하는 메소드인지를 알아야 원하는 부분을 수정할 수 있으므로 상속받은 메소드의 구조를 알고있어야한다.

view함수에서 무조건하나의 파라미터를 선언해야되는데 그게 request이다. request는 요청사항을 의미한다
요청파라미터와 역할은 같은데 path 파라미터를 추가로 받을 수 있다.
ex) id,pwd입력하고 로그인을 누르면 뭐를 입력했는지 db에서 확인을 하는데 이 값들을 가지고 가야한다. 그래서 id, pwd같은 경우를 요청 파라미터라고하고 요청 파라미터는 ()이다. get방식은 요청정보를 url뒤에 붙여서 보내고 post방식은 url이 아닌 body에 요청정보를 넣어서 보낸다. 
경로에 파라미터를 붙인것을 path parameter라 한다. => 목적은 요청 파라미터와 같지만 전달하는 방식이 다르다.
path parameter가 옵션인 이유는 없을 수도 있기때문이고 여러개를 받아줄 수도 있다. 요청 파라미터는 ?를 구분자로 사용했지만 path parameter에는 없다. 이거를 urls.py에서 처리해줌

path("/detail", view.post_detail)
https://nid.naver.com/detail
이라는 요청이오면 post_detail()함수를 불러라

path("/detail/<int:post_id>", view.post_detail)
https://nid.naver.com/detail/3
