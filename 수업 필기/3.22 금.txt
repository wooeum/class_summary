외부조인은 한쪽을 존재하든 안하든 다 보고싶은 경우에 사용한다.

서브쿼리는 행이 1000개일경우에는 1000번 실행된다

하나의 작업에서 여러 db가 바뀌는 경우에는 다 바뀌거나 다 안바뀌거나
이러한 작업을 한 단위로 트랜잭션이라고 한다. 트랜잭션 처리.

하나만하고 나머지가 안되면 처음으로 돌아가라 =>rollback
끝까지 끝나면 1개의 작업이 끝났으므로 => commit했다

오토커밋은 커밋처리를 db가 알아서 해준다.
근데 db가 이러한 작업단위를 알까? 모른다. 쿼리문 하나마다 계속 처리를 해주는게 오토커밋 
우리가 롤백할지 커밋할지 수동으로 정해주는게  매뉴얼 커밋

insert -> insert -> insert -> rollback => 처음으로 돌아감
insert -> insert -> insert -> commit => 일이 끝남

인서트,딜리트,업데이트는 하나의 쿼리에 하나의 일을함
테이블단위로 일을 함

다 컬럼단위인데 얘는 행단위. 그래서 컬럼을 지정해주지 않아도 됨. 컬럼을 없애려면 널값을 주면된다
