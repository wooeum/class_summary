결국 웹이라는건 http 프로토콜에서 돌아가는거다.
옛날에는 프로그램과 데이터 모두 내 컴퓨터에 있어서 db나 파일에 저장해서 사용했지만
다른 컴퓨터에 있는 자료를 쉽게 보기위해 컴퓨터를 연결한 것이 네트워크가 되었다.
우리가 기본적으로 두개의 컴퓨터를 연결했다하면 한쪽에는 데이터가 있고 다른 한쪽은 그 데이터를 사용하고 싶을 것이다. 데이터를 제공하는 쪽은 서버 사용하는 쪽은 클라이언트가 되는것이고, 데이터를 사용하기 위해서 연결이 되어있어야하고 프로그램을 사용해야한다. 클라이언트가 뭐를 한다하면 네트워크를 타고 요청을 보내고 서버는 요청에 대한 처리 결과를 보내주는 것이다. 프로토콜이란 어떤 방식으로 데이터를 주고받는지를 정해놓은 규약이고 http 프로토콜도 여기에 해당한다.
그래서 우리가 요청할때 요청 방식이라거나 요청정보, 응답방식들이 모두 프로토콜에 정의되어있는것이다.
GET방식, POST방식, 요청을 처리하는 방식 등을 알고있어야한다. 


MVT 패턴: 한 파일에 다 집어넣는데 역할에 따라 나누겠다.
요청이 들어오면 요청에 대한 처리를 하고 요청이 끝날것이다. 이 과정에서 뭐가 반복되는지를 보니까 DB관련된게 많이 반복되고 응답메세지도 자주 반복된다.(1. 중복적이거나  2.코드를 복잡하게 만드는것)
한번에 메서드하나 실행해서 select를 실행하는게 아니라, 커리문 하나를 실행하기 위해선 마우스 가져오고 커서 ~ 등등의 메서드가 많이 필요하다. 그러므로 따로 만들어서 호출하는 방식으로 바꾸자.
Model, View, Template DB처리, 흐름관리, 응답처리.
